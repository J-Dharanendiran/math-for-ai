# 🧩 Reading the Graph's Language — Piecewise Functions

---

> *"Some stories don't follow one rule all the way through. Neither do some functions."*

---

You've already seen how a single equation like **y = |x − h| + k** can describe a V-shaped graph — one rule, applied everywhere. But what happens when a graph *refuses* to follow one rule? What happens when the function behaves one way for a while, then jumps and behaves completely differently?

That's not a broken function. That's a **piecewise function** — and it's one of the most honest ways mathematics has ever described the real world.

---

## 📺 What You're Actually Looking At

The graph in the video shows horizontal lines that sit flat for a stretch, then *jump* to a new height, then jump again. It looks like a staircase drawn by someone who couldn't decide on a direction. Mathematically, this is called a **step function** — a special case of piecewise functions where each piece is just a constant.

But don't let the simplicity fool you. The *idea* being demonstrated — that a single function can use different rules for different regions of the x-axis — is the same idea powering ReLU activation, hinge loss, tax brackets, and signal thresholds. The step function is just the clearest possible window into that idea.

Think of it this way: most functions you've seen so far are like *one singer performing the entire song*. A piecewise function is more like *a relay race* — each runner takes over for their stretch, and there's a precise handoff point where one stops and the next begins.

## 🏁 The Starting Point — Breaking the X-Axis into Territories

The very first move when reading any piecewise graph is to *stop looking at the y-values* and instead ask: **where does the rule change?**

Those change-points become the boundaries of your intervals. In the video's graph, the rule changes at x = −5 and x = −1. So the x-axis gets divided into three territories:

- Everything from just after **−9** up to and including **−5**
- Everything from just after **−5** up to and including **−1**
- Everything from just after **−1** up to and including **9**

Each territory gets its own rule. That rule, in a step function, is simply the height of the flat line in that region. Once you've identified the territories and their heights, you have the complete function — you just need to write it down properly.

---

## 🔵 The Open Circle Problem — Where Most Students Make the Critical Error

Here's where the story gets careful. Look at the graph at x = −9. There's an *open circle* — a hollow dot. At x = −5, there's a *closed circle* — a filled dot. This is not decorative. This distinction carries the entire mathematical weight of the function.

> **An open circle means the function does not include that point. A closed circle means it does.**

So when you write the first interval, you're saying: the function is active starting *just after* −9 (open, excluded) and runs all the way *through* −5 (closed, included). In inequality notation:

$$-9 < x \leq -5$$

The function value across this entire stretch is the constant **−9**, so the first piece is:

$$f(x) = -9 \quad \text{if } -9 < x \leq -5$$

The second territory starts just after −5 (open, since −5 was already claimed by the first piece) and runs through −1 (closed). The height here is **6**:

$$f(x) = 6 \quad \text{if } -5 < x \leq -1$$

And the final stretch: just after −1 (open) through 9 (closed), sitting at a constant height of **−7**:

$$f(x) = -7 \quad \text{if } -1 < x \leq 9$$

Put them all together, and this is the complete function:

$$f(x) = \begin{cases} -9 & -9 < x \leq -5 \\ 6 & -5 < x \leq -1 \\ -7 & -1 < x \leq 9 \end{cases}$$

---

## ⚖️ The One Rule That Cannot Be Broken

The boundary logic above isn't arbitrary fussiness — it comes from the most fundamental rule in all of mathematics:

> **A function must give exactly one output for every input. No input can produce two different outputs simultaneously.**

At x = −5, two pieces meet. If *both* used ≤ at that boundary, then x = −5 would return **both −9 and 6** at the same time. That's not ambiguous — it's *invalid*. The graph would fail the vertical line test. It would stop being a function entirely.

So the rule is clean and non-negotiable: **at every shared boundary, exactly one piece owns that point.** The other piece must use strict inequality (`<`) to exclude it. One closed circle, one open circle. Every time.

This is a checkpoint worth drilling into your instincts. Before writing any piecewise function, ask yourself at each boundary: *who owns this point?* If both intervals claim it, you've broken the definition of a function.

---

## 🧠 The Mental Process — A Repeatable Checklist

Reading a piecewise graph well is a skill that becomes automatic with practice. The sequence always follows the same four steps:

1. **Scan the graph for rule-change points** — wherever the graph jumps, ends, or changes its behaviour is a boundary.
2. **Write the interval for each piece** — use the open/closed circle rule to decide `<` vs `≤`.
3. **Identify the formula** — in simple cases this is a constant height. In harder cases it could be a line, a parabola, or an exponential.
4. **Assemble the piecewise notation** — combine all pieces into the brace notation, one row per interval.

The direction must work both ways. You should be able to go *from graph to equation* and *from equation to graph* with equal fluency. If you can only do one, you're only half-literate in the language.

## ⚡ The Same Structure Powers Your Neural Networks

This is where the story earns its relevance.

**ReLU**, the activation function used in nearly every modern neural network, is a piecewise function:

$$\text{ReLU}(x) = \begin{cases} 0 & x < 0 \\ x & x \geq 0 \end{cases}$$

It returns zero for all negative inputs, and the input itself for all non-negative inputs. That's two territories, two rules, one shared boundary at x = 0. When you learned to identify the closed circle at x = −5, you learned the same reasoning used to define whether ReLU *includes or excludes* the boundary at zero.

**Hinge loss**, used in Support Vector Machines, is also piecewise:

$$\mathcal{L}(x) = \begin{cases} 0 & \text{if } yx \geq 1 \\ 1 - yx & \text{if } yx < 1 \end{cases}$$

**Huber loss**, which combines L1 and L2 loss into a hybrid that's robust to outliers:

$$\mathcal{L}_\delta(x) = \begin{cases} \frac{1}{2}x^2 & |x| \leq \delta \\ \delta(|x| - \frac{1}{2}\delta) & |x| > \delta \end{cases}$$

Three different loss functions, three different design choices — but all sharing the same skeleton. Understand how to read and write piecewise notation, and you can read the architecture of these loss functions directly, rather than treating them as black-box formulas handed to you by a textbook.

## 📊 Real World — Tax Brackets as a Piecewise Function

The most tangible example lives in every tax system on earth. Income tax in most countries works like this: you pay one rate on the first portion of your income, a higher rate on the next portion, and so on. That is *exactly* a piecewise function.

A simplified version looks like:

$$\text{Tax}(x) = \begin{cases} 0.05x & 0 \leq x \leq 50{,}000 \\ 2{,}500 + 0.20(x - 50{,}000) & 50{,}000 < x \leq 150{,}000 \\ 22{,}500 + 0.30(x - 150{,}000) & x > 150{,}000 \end{cases}$$

Notice: the boundaries are carefully handled so no income level falls into two brackets simultaneously. The same open/closed circle logic, just without the circles — enforced by the inequality signs.

> *"A step function in a math class is a tax bracket in real life. The mathematics doesn't change. Only the stakes do."*

## 🔬 A Deeper Look — Continuity and Discontinuity

Something worth understanding beyond the mechanics: the step function in this video is **discontinuous**. At x = −5, the function *jumps* from −9 to 6. There's no smooth transition. If you tried to draw this graph without lifting your pen, you couldn't.

Not all piecewise functions are discontinuous. Consider a piecewise function where the pieces connect smoothly at the boundaries:

$$g(x) = \begin{cases} x + 1 & x < 2 \\ 2x - 1 & x \geq 2 \end{cases}$$

At x = 2: the left piece gives 2 + 1 = **3**, and the right piece gives 2(2) − 1 = **3**. They meet at the same point. This function is *continuous* — even though it's defined piecewise.

This distinction matters in calculus. A continuous piecewise function might be differentiable at its boundary (if the slopes also match). A discontinuous one like the step function is neither continuous nor differentiable at the jump. That's not a problem — it's a property, and knowing it tells you what you can and can't do with the function analytically.

## ⚠️ The Mistakes That Betray Shallow Understanding

These are not careless errors — they're signs that the concept hasn't fully landed yet:

- **Using ≤ on both sides of a shared boundary.** This is the most common mistake and it breaks the definition of a function entirely.
- **Reading the y-value at the wrong interval.** The height of the graph tells you the *output* — but only if you've correctly identified which interval you're in.
- **Forgetting to check the boundary ownership.** At every transition point, explicitly ask: which piece claims this value?
- **Assuming all piecewise functions must be step functions.** Each piece can be *any* function — linear, quadratic, trigonometric, exponential. The step function is just the simplest possible case.
- **Treating piecewise notation as a formatting trick.** It's not notation — it's a complete mathematical definition. The curly brace is saying: *the function is entirely described by what follows*.

## 🏋️ Drill Problems — Both Directions

Work these until the process feels mechanical:

1. The graph has a closed circle at (−3, 4) and an open circle at (2, 4) for the first piece, then an open circle at (2, −1) and a closed circle at (7, −1) for the second piece. Write the complete piecewise function.

2. Write f(x) as a piecewise function if its graph shows: a line y = x + 2 from x = −4 (open) to x = 1 (closed), then a constant y = 3 from x = 1 (open) to x = 5 (closed).

3. Given the function:
$$f(x) = \begin{cases} x^2 & x < 0 \\ 2x + 1 & 0 \leq x \leq 3 \\ 7 & x > 3 \end{cases}$$
Evaluate f(−2), f(0), f(3), and f(5). Which piece does each input use?

4. Is the function in problem 3 continuous at x = 0? At x = 3? Check by evaluating the limit from each side.

5. Write a piecewise function that is *continuous* at x = 2, where the left piece is y = 3x − 1 and the right piece is a constant. What must that constant be?

6. *(Hard)* Show that ReLU(x) = (x + |x|) / 2. Use this to connect what you learned about absolute value directly to the piecewise definition of ReLU.

## ✅ What This Unlocks

> *"The student who can write any graph as an equation — and any equation as a graph — has the entire coordinate plane as a language."*

Piecewise functions are not a detour. They are the *honest form* of most functions that model real behaviour. Temperatures don't follow one formula all day. Economic systems don't obey one equation across all income levels. Neural networks don't use one activation everywhere.

The step function in the Khan Academy video is elementary. But the thinking it demands — *break the domain into regions, assign each region its own rule, handle boundaries without overlap* — is identical to the thinking used when designing loss functions, writing conditional logic in code, or analysing any system that behaves differently in different regimes.

Master the simple case completely. The complex cases are just more pieces.
